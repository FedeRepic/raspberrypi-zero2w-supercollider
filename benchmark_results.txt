--- Performance Optimization Tips ---

1. Always use "performance" CPU governor for audio work
2. Use blocksize 512 or 1024 for stability
3. Reduce blocksize to 256 for lower latency (watch for xruns)
4. Avoid SD card I/O during audio playback
5. Use external USB audio interface (ES-9) for best quality
6. Keep SuperCollider code optimized (avoid unnecessary allocations)
7. Monitor CPU usage with s.avgCPU and s.peakCPU
8. Disable unused services to free up RAM
9. Consider using swap file on USB drive (not SD card)
10. Test your patches incrementally to find CPU limits
